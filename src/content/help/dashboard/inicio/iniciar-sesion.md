## Antes de empezar

Ten a la mano el correo con el que te dieron acceso al Dashboard. El acceso depende de que tu usuario este activo y asignado al local correcto. Si trabajas con varias sucursales, el mismo usuario puede ver mas de una, pero cada una puede tener permisos distintos.

Tambien confirma que estas entrando al Dashboard Web, no a una herramienta interna o a una liga de pago publica. La pantalla correcta debe pedir correo y contrasena de usuario.

## Pasos

1. Abre la URL oficial del Dashboard.
2. Escribe tu correo completo, sin espacios al inicio o al final.
3. Escribe tu contrasena.
4. Si el navegador ofrece guardar la contrasena, confirma que sea para el dominio correcto.
5. Al entrar, revisa el nombre del local activo en el selector de sucursales.
6. Si esperabas otra sucursal, cambia de local antes de revisar pagos, pedidos o reportes.

## Problemas frecuentes

Si la contrasena no funciona, usa recuperacion de contrasena antes de pedir un nuevo usuario. Si entras pero no ves una seccion, el problema normalmente es de permisos, no de inicio de sesion. Si no aparece ninguna sucursal, pide a un OWNER o ADMIN que revise que tu usuario este asignado al local y que siga activo.

## Validaciones de acceso

Despues de entrar, confirma tres cosas: correo de usuario, sucursal activa y secciones visibles en el menu. Si el usuario trabaja en varias sucursales, cambia de local y revisa si el menu se actualiza. Esto ayuda a separar problemas de sesion de problemas de permisos.

Si vas a compartir instrucciones con un nuevo miembro del equipo, dile que use siempre el dominio oficial del Dashboard y que no comparta contrasenas. Para roles operativos, conviene probar el acceso antes del turno para evitar bloqueos durante servicio.

## Cuando pedir ayuda

Pide soporte si el usuario no recibe recuperacion de contrasena, si el login entra en ciclo, si aparece una pantalla sin sucursales despues de confirmar asignacion o si Google Sign-In falla en el dominio correcto. Incluye correo del usuario y sucursal esperada.
