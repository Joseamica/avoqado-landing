## Antes de empezar

Antes de publicar una promocion, define objetivo, vigencia, restricciones y canal donde se comunicara. Distingue entre descuento y cupon: un descuento puede ser una regla operativa; un cupon suele depender de un codigo o condicion especifica.

Confirma tambien si la promocion aplica a todos los productos o solo a categorias, clientes o periodos concretos. Esto evita descuentos no deseados en caja o reportes confusos.

## Pasos

1. Abre **Clientes > Promociones**.
2. Entra a **Descuentos** para reglas de descuento.
3. Entra a **Cupones** para codigos promocionales.
4. Revisa nombre, valor, vigencia, restricciones y estado.
5. Guarda y valida que la promocion aparezca activa o programada.
6. Antes de comunicarla, prueba el flujo operativo que usara el equipo.

## Problemas frecuentes

Si una promocion no aplica, revisa vigencia, estado, restricciones y sucursal. Si aplica cuando no deberia, desactiva la regla y revisa condiciones. Si el equipo no la ve, confirma permisos y canal de venta. Si hay diferencias en reportes, identifica si el monto aparece como descuento, cupon o ajuste.

Para campanas sensibles, deja una nota interna con responsable, fecha de inicio, fecha de cierre y criterio de exito.
